# Repository Audit

Дата аудита: 2026-06-18

## Current Architecture

`src/russian_markets_lab/moex_client/`  
Клиент и функции доступа к публичным MOEX ISS таблицам: securities, candles, marketdata, trades, order book best-effort, futures, options.

`src/russian_markets_lab/analytics/`  
Чистые аналитические функции: market universe, liquidity, volatility, futures basis, options Greeks/surface, portfolio risk, execution cost modeling.

`src/russian_markets_lab/backtester/`  
Векторизованный research backtester, portfolio utilities, costs, metrics.

`src/russian_markets_lab/strategies/`  
Research templates для momentum, mean reversion, stat arb, carry и regime filters. Это не trading signals.

`src/russian_markets_lab/dashboard/`  
Streamlit frontend. До hardening-а содержал часть бизнес-логики, fallback returns и sample equity curve прямо в UI.

`src/russian_markets_lab/reports/`  
HTML report builder для liquidity и derivatives/risk reports.

`src/russian_markets_lab/scripts/`  
CLI-style script `build_research_snapshot.py`. До hardening-а был слишком большим и совмещал ingestion, transformations, persistence и report building.

`tests/`  
Unit tests для core math, MOEX parser, execution, risk, costs, backtester. Требуется расширение на provenance/cache/pipelines/dashboard helpers.

`notebooks/`  
Семь notebook-шаблонов. До hardening-а были mostly markdown placeholders, не полноценные executable research notebooks.

`data/`  
`data/processed/` содержит parquet snapshots. `data/raw/` пока не использовался как полноценный raw cache.

`reports/`  
Сгенерированные HTML reports.

`assets/`  
Dashboard screenshots.

## Data Sources

### MOEX ISS derived outputs

- `data/processed/market_universe.parquet`
- `data/processed/liquidity_radar.parquet`
- `data/processed/futures_basis.parquet`
- `data/processed/options_chain_features.parquet`

Эти outputs строятся из публичных/задержанных MOEX ISS endpoints для TQBR shares и FORTS futures/options.

### Processed parquet outputs

Dashboard and notebooks read from `data/processed/*.parquet`.

### Cached outputs

Before hardening, only optional JSON cache existed in `MOEXISSClient`. A structured raw parquet cache under `data/raw/<dataset_name>/` was missing.

### Demo/sample/fallback outputs

Occurrences found before hardening:

- `src/russian_markets_lab/dashboard/app.py`: hardcoded `SBER/GAZP/LKOH` returns in risk tab.
- `src/russian_markets_lab/dashboard/app.py`: sample equity curve in strategy tab.
- `src/russian_markets_lab/scripts/build_research_snapshot.py`: `_fallback_universe` built a one-row-observation universe from current marketdata when candles were unavailable. This is a transparent fallback from real marketdata, but metadata/provenance was missing.
- Tests contain mocked SBER/GAZP values; this is acceptable because tests are not production output.

### Manually generated outputs

- PNG screenshots in `assets/` were manually captured from Streamlit.
- HTML reports in `reports/` were generated by local pipeline.

## Hardcoded/Demo Data Occurrences

Allowed:

- `tests/test_moex_client.py`: mocked MOEX table rows with `SBER`, `GAZP`.
- Test fixtures and mocked parser payloads.

Needs refactor:

- Dashboard risk fallback returns for `SBER`, `GAZP`, `LKOH`.
- Dashboard strategy sample equity curve.
- Any silent fallback that looks like real research output without metadata.

## Architecture Problems

- Dashboard is not thin enough: it mixes UI with risk calculations and fallback demo series.
- `scripts/build_research_snapshot.py` is too large and should become an orchestrator over pipeline modules.
- Processed datasets lack JSON metadata sidecars.
- Raw cache exists only partially through optional JSON request caching, not as raw table parquet snapshots.
- Notebooks are readable but not executable research stories.
- Reports need richer metadata and separate execution/project overview reports.
- Tests are strong for math, weak for provenance, raw cache, pipeline boundaries and dashboard helpers.
- Methodology, data sources and limitations docs are missing.

## Immediate Priorities

1. Remove silent fake/demo data from production dashboard.
2. Add explicit opt-in demo mode with visible warnings.
3. Add processed dataset metadata sidecars.
4. Implement raw cache with parquet + metadata.
5. Split `build_research_snapshot.py` into reusable pipeline modules.
6. Refactor dashboard into `data_loader.py`, `charts.py`, `components.py`.
7. Add CLI for dataset builds and dataset status.
8. Add methodology/data sources/limitations docs.
9. Upgrade notebooks into executable research notebooks.
10. Add tests for metadata, IO, cache, pipelines, dashboard helpers and no-fake-production-data scan.
